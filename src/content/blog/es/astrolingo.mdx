Tengo un **nuevo sitio web**. ¡Este que estás viendo!

Es **rápido**, **sostenible**, y completamente **multilingüe**.

## Veloz como un rayo (y creado con Astro)
Lo construí con [Astro](https://astro.build/), un **marco web open-source que enfatiza la velocidad/rendimiento**. Empresas como [Porsche](https://www.porsche.com/), [NordVPN](https://nordvpn.org/), [Firebase](https://firebase.blog/) y [Netlify](https://www.netlify.com/) usan Astro gracias a su excelente rendimiento.

Y es que la **velocidad** y el **rendimiento** de un sitio web es **cada vez más importante** pues:
- ayuda a [reducir las tasas de rebote](https://www.cloudflare.com/learning/performance/why-site-speed-matters/), [mejorar las clasificaciones SEO](https://www.edgeoftheweb.co.uk/blog/importance-of-page-speed) y [la conversión](https://developer.mozilla.org/en-US/docs/Learn/Performance/why_web_performance)
- y también está [estrechamente relacionado con la sostenibilidad de un sitio web](https://www.reffine.com/en/blog/Page-load-speed-vs-sustainability-correlation-between-them).

Los sitios web de Astro tienen un rendimiento extraordinario porque envían menos código al navegador que otros marcos como React, Preact, Vue y Svelte.

Esto no quiere decir que vea a Astro como un competidor de React o cualquiera de esos otros. Parte de lo que me gusta de Astro es que permite combinar componentes escritos en otros frameworks cuando hay partes del sitio que requieren interactividad. Por ejemplo, la animación en esta página web está construida con React y Three.JS.

Por tanto, no veo a Astro como un reemplazo para otros frameworks sino como una forma de combinar lo mejor de diferentes opciones.

## Multilingüe

El sitio también está construido encima de un tema Astro open source que creé llamado [**AstroLingo**](https://github.com/jbolns/astrolingo).

AstroLingo tiene como objetivo facilitar la gestión de sitios web multilingües Astro. Los sitios web multilingües requieren mucho trabajo que a veces no es obviou debido a la necesidad de enrutar automáticamente el contenido a través de secciones e idiomas. Si necesitas o quieres un sitio, blog o portafolio multilingüe, *AstroLingo* puede ahorrarte muchas horas de trabajo; días, incluso.

¡Y funciona! ¡Sólo mira este sitio!

El reto ahora es mantener el ritmo de creación de contenidos. Los sitios web multilingües requieren mucho contenido. Ya veremos cómo va la cosa.
